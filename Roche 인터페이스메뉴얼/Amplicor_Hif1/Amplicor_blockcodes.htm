Composition of the blocks

Please find block composition examples in the section `Error! Reference source not found.. Error! Reference source not found.'.

The description of the symbol notification used can be found in the section
`2.2. How to use this manual'.

Block 00: Sync Block / Idle Block

Block 00 is used during the synchronization procedure (refer to section `2.6.3. Synchronization between Host and COBAS Instrument') or as an answer to a result request if no results are available.

Block 00 has no lines.

Sync block configuration: no block check
Idle block configuration: block check included

Block 04: Result List Request

The Result List Request Block is used to get a list of A-ring Id's which have at least one unsent result.

Block 04 has no lines.

Block 07: Result List

Response to a Result List Request. The Result List Block contains all A-ring Id's which have at least one unsent result.

Block structure :

x = 1..200
y = 0 or 1 (dependent on result list request)

Block 08: Result Data

The Result Data block is produced by the COBAS Amplicor in response to a Result Request block sent by the Host. This Result Data block consists of at least one particular result (one A-tube test result)!

If several results for one A-ring ID are available, they are transmitted within one result block.

The configuration of the Result Data block will correspond to the Result Request. The following block structure is used:

Qualitative Result:

x = 1..6

max. 6 qualitative tests per A-tube

y = 1..12

max. 12 A-tubes per A-ring

z = 1..2

max. 2 quantitative tests per A-tube

After an occurrence of an error in a Result Request block or Result List Request block, the COBAS Amplicor transmits a Result Manipulation Response block, which contains the error code line.

Block structure : Error code: Please refer to Line Code 99 description.

Block 10: Order Entry

The Order Entry Block 10 allows the creation of new orders, the addition of an A-tube and the respective tests to an existing A-ring or the addition of tests to an existing A-tube.

At least one test for at least one A-tube must be ordered.

· If the Order Date/Time Line 02 is omitted, the system time of the COBAS Amplicor will be used instead.

· If the Order Info Line 06 is added, the same information will be sent back to the Host within the Result Data Block 08.

· If the Test Condition Line 08 is omitted, the following default values will be used:

_ Tests: Normal Test

_ Controls: Positive Control

_ Dilution Tests: Always a normal test is performed, i.e. line 08 is disregarded

_ Dilution Controls: Positive Control

y = 1..12 · max. 12 A-tubes per A-ring

1 · if quantitative tests are ordered

Block 19: Order Manipulation Response

Each request with block code 10, 11 or 12 is answered by block 19, the Order Manipulation Response. The block content is dependent on the request.

The error code in Line 99 and the response message structure depends on the success of the request and also on the kind of request. One of the following block structures will appear:

General order manipulation response:

Response to correct order manipulation
_ Error Code 00:

Response to order manipulation with a
general error _ Error Code XX:
Please refer to Line Code 99 description.

Order manipulation response reporting an A-tube fault:

A-tube faults that arise during an order manipulation will be reported with the corresponding error code and an additional line 04 (A-tube position). Nevertheless, all order manipulations without error will be executed!

Order extension: y = 1..11
The maximum value is the same as maximum A-tubes per A-ring. Order extension is only possible if one tube test order exists already _ 11 tubes.

Order deletion: y = 1..11
The last A-tube of each order cannot be deleted.

Order manipulation response reporting a test fault:

Every test error that arises during an order modification will be reported with the corresponding error code and the required additional information. However, all other requested manipulations will be executed!

Order extension: z = 1..71 / 1..23

The maximum value is the same as maximum A-tube tests per A-ring. (6/2 tests per A-tube, 12 A-tubes per A-ring.

Order extension is only possible if one tube test exists already _ 71/23 tests).

Order deletion: z = 1..60
The last A-tube test order of each A-tube cannot be deleted
(_ max. 5 tests per tube _ max. 60 tests in total)

Block 90: State of Instrument Request

The State of Instrument Request block is used to get information about the actual status of the instrument.

The status selection line 00 specifies to send only system states or send additional load information (A-ring's, reagent cassettes and D-cup racks).

Block structure: Service:
0 : System State
1 : Detailed System State
2 : Detailed System and Load State

Block 91: State of Instrument Data

Each request with block 90 State of Instrument Request is answered by block 91 State of Instrument Data. The block 91 content depends on the request.

This block contains information about the system state, TC states, DP states and if selected load status (A-rings, Reagent racks and cassettes).

Response to a System State request:

Response to a Detailed System State request:

Response to a Detailed System and Load State request:

If the Run Status in line 90 > 2 then w=2

x = 0..4 : Number of currently loaded A-rings

y = 0..3 Number of currently loaded Reagent Cassettes

z = 0..3 Number of currently loaded D-cup racks
