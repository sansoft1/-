800 Host Interface Known issues

Collated results

  • Due to instrument performance reasons and in order to have results on the Host system available as soon as possible,  we strongly recommend NOT using "Collated Results" requests ( i.e., not using Result Type Selector 03, 04, 05 and 06 ).
  • Ratios will not be collated with other results, as they have automatically a different sample type.
  • Collated results selector 04 transmits also blocked and invalid test results together with valid ones. Although they are clearly flagged as such, this may become an issue if the host does not properly handle result flags.

Empty orders

  • Block type 10 having no tests (no line code 55 present) have been reported to hang the host interface on the INTEGRA 800. 
    This behavior could not be reproduced and probably happens at random. This situation is triggered by an unsupported configuration of the order entry block an should be avoided from the host side.

Timing and priority of order entry.

  • The automatic handling of RD 5 racks in the INTEGRA 800 creates a special situation regarding Host interface information flow: 
    New inserted RD 5 racks remain only for a limited time on the instrument. see Workflow adaptation 

    The first priority should always be the acquisition of new results

    When lists of "tubes without orders" are returned from the instrument, uploading of the order entries has to be given priority over patient entries.
    If patient entries are given a higher priority, over orders,  this may lead to the situation where some racks time out, before receiving orders and are therefore ejected on the output tray without tests performed.
