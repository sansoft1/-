*Patient Mode

- Patient information is only required when in patient mode: ( To set up patient mode, see 700 / 800 / 400 )
- When in patient mode: prior to an order entry, set up the patient on the instrument.
- When connected to a host, we recommend working in sample mode.

Warning: 
( applies only if in Patient mode )
     Patient ID has to be unique. 
>> Do not use the same Patient ID for more than one patient, otherwise results could be assigned to the wrong patient !!

Patient Entry

Patient information is required by the Integra only when the instrument runs in Patient Mode.( 400 / 700 )
Patient information is downloaded by means of BC40, Patient Entry.

 

Patient Modification

Patient information is modified by means of BC42, Patient Modification.

 

Patient Deletion

Patient information is deleted by means of BC41, Patient Deletion.
Patient deletion is possible only if there are no orders attached to it.

 

Order Entry

Single tests, ratios or profiles can be ordered using an Order Entry (BC10).
An Order Entry may contain up to 50 attached tests.
When using ‘Patient Mode’ (400 / 700), an order can only be entered if the patient information is already available in the INTEGRA’s database.

Note: Integra 700 only, Profiles and Ratios are supported only by SW version 9662 and higher.

 

Order Modification (Add Tests)

Adding tests to orders which already exist on the COBAS Integra is done by means of an Order Entry (BC10) without the Order Information line (LC54).

 

Order Deletion

Orders are deleted by means of BC11, Order Deletion.

Order deletion is possible if
- the order either has not yet been started, or
- the order is finished and all results are accepted.

"Host Query" Mode

Host Query Mode is the most convenient way to work with laboratory systems. It allows the user to put samples on any instrument at any position on any sample rack. It optimizes Laboratory workflow, particularly when the laboratory has more than one Integra.

A barcoded sample is placed in an instrument. The Instrument reads the barcode and waits for a Host request. The Instrument then sends a list of any on-board samples lacking order information.

Due to the Master Slave principle, the Host MUST transmit an MCS request (BC60) in order to receive the COBAS Integra response containing sample information. To ensure an optimal Host Query function, the Host system has to transmit this request periodically, e.g. one request every 30 seconds.

Following an MCS request, a complete list of all pending sample tubes lacking test orders is transmitted to the Host system (all tubes are listed within the Block 62).

Every entry in this list, i.e. every reported pending sample tube, requires its own order entry block (Block 10).

The Host system should ignore pending sample tube information if it does not have associated order information available. The tubes will be listed with every COBAS Integra response to a Host MCS request until they are removed from the Instrument.

To reactivate the Host Query for a sample (in order to allow reruns), the Host should always transmit an Order Deletion block (Block 11) as soon as it has received all results belonging to a specific order.

Note:
Requests for samples without order lead to extensive database access.
Dependent on the size of the database and the type of request this may result in poor instrument performance, i.e. keep the Host’s request intervals as long as possible. (e.g. 60 seconds)		 !

 

 

Rerun Tests

Unlike the user interface, the Host interface does not support order reruns. Reruns are performed by again adding the required test to the existing order.

 

 

Result Request

Due to the Master-Slave principle, results are transmitted to the Host only upon the receipt of a Result Request (BC09).

Cobas Integra allows requesting single results or collated results. The BC09/LC10 thus contains a result type selector.

We strongly advise against the use of collated results, depending on the database size or fragmentation, we cannot guaranty the instrument performance.

No specific result can be requested.

Only accepted results will be transmitted.
(Auto accept can be configured in the ‘Configuration / Processing / Result Handling Table’ menu.)

Usually each result will be transmitted only once. To re-transmitted a result:
- On the COBAS INTEGRA user interface, change to the result workbench.
- Select the required results with the mouse.
- Select ‘Retransmit’ in the  ‘Process’ menu.

We recommend using the following procedure and timing for the result request loop:

Note:
Result requests lead to extensive database access.
Dependent on the size of the database and the type of request this may result in poor instrument performance, i.e. keep the Host’s request intervals as long as possible. (e.g. 60 seconds)		 !

 

We strongly recommend sending an Order Deletion Block (BC11) to the Integra as soon as all results associated with an order have been transmitted to the Host. This provides several advantages:

- Re-entered samples will trigger the Host Query again
- The Order- and the Result-screen will show only data of current interest.
- The Integra’s Data Station performance will be improved.
- No manual data maintenance is required on the Integra .

By using this procedure, Order and Results-Screens are cleared. On the Integra 700 the results are still available from the Patient Archive (raw data will be deleted). On the Integra 400 all data are deleted.

 

 

Using Samples with Barcode Identification

If barcodes are used to identify samples, the barcode includes the order number and can also include the sample type. (‘Barcode Set-up’ Integra 400 / 700).

When using barcodes, both the parameters ‘Sample Position Rack Number’ and ‘Sample Position Tube Number’ have to be set to 0 (zero) in order entries (see LC54).

 

Protocol Version

The Protocol Version request (BC92) is used to get the Version # of the HIF Protocol.
