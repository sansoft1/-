Line Code 53: Order ID

 

Description

Format

Range

Line Code

53

  

Order number

S15

 Integra 400: Barcode characters
Integra 700 / 800: ASCII 33 to 126

Order date

D1

 01/01/1800 to 31/12/2100 and 00/00/0000 = ’today’
If Sample Type is used
Sample type S3 Laboratory defined string
Integra 400: Barcode characters 
Integra 700 / 800: A to Z , 0 to 9 , _ , ??? , ***
For configuration, see Integra  400 / 700 / 800

Example: 53_abcdef987654321_00/00/0000_URI<LF>

Note: The order number and the sample type may not contain <space>s in front of or within the identifier. Nevertheless, if the fields are not used in their full length, digits left over have to be filled with <space>s.

Remark: There are three generally pre-defined sample types:

??? : Sample type determined by Integra's test definitions. Used with order entries and deletions if the test-specific sample type should be selected automatically.
*** : These wild cards are used with order deletions to select all sample types.
RTO : Integra sets sample type to RTO for ratio results.
Warning: 
Using a sample specific code instead of ??? , (for example HEM for hemolysate) will override any instrument and test default sample settings and force ALL tests from the order to be executed on such a sample type.  
This has the same effect as selecting a specific sample type on the order entry screen of the instrument, instead of "automatic", which is the default in the drop down list.
Note: .
Integra 700 / 800: If the database set-up parameter ‘Sample Types Used’ is disabled, all sample types received by the Integra are ignored . Also no sample type information is transmitted to the Host.
. .
Integra 400: Orders: If the Sample Type is not transmitted, the default sample type (defined by test or rack definition )  is used.
. Results: If in ‘Configuration Processing / Result Output / Sample Type’,  the sample type is disabled, no sample type information is transmitted to the host.

 

Note regarding Ratio Orders:
If the component results of a ratio are measured from different sample types, the 'Order Entry's Sample Type must be '???'.
As described in the know issues: ratio results are not collated when collated results result requests are used.
