700 Instrument Set-Up
1. Prepare an interface cable and connect the cable to the Host and the Instrument
2. Enable Host interface transmission (Config. / Processing / Result Output Mode)
3. Set-up interface parameter (Config. / General / Host Interface)
4. Prepare the database set-up (Config. / Lab / Database Set-up)
5. If you are using barcodes, set-up the instrument accordingly.
6. If you modified anything, shutdown and restart the Data Station
7. Load your application, e.g. LIS or terminal program. As a first check send an idle block

Enable Host interface transmission

 

The COBAS INTEGRA’s Host Interface is enabled/disabled in the ‘Configuration / Processing / Result Output Mode’ menu.

Parameter

 Value

Description

Transmission

 Off

The Host interface is disabled. The instrument does not allow any request from the Host and no responses are generated.
           On

The Host interface is enabled and functions in accordance with the definitions.

Data Selection

 Samples Only

Only patient results will be transmitted in response to a result request.
           All Data

Every kind of result (sample, calibration, control) will be transmitted in response to a result request.

Important Note: Because of an error in the Hewlett-Packard Data Station, the parity check and the hardware handshaking do not function properly. If parity even or odd is selected, no data will be transmitted. Therefore:
On COBAS INTEGRA always select a parity of none and always select software handshake.

Note: All interface parameters have to be set to the same value on both Host and COBAS INTEGRA 700.

The following items are configured in the  ‘Configuration / Lab / Database Set-up’menu.

Parameter

Value

Description

Usage Mode

Patient

All sample orders have to be assigned to an existing patient. The instrument generally works with patient data.
         

Sample

The instrument generally works without patient data. Orders have to be entered without patient data.

Sample Types Used

No

The instrument generally works without sample types. No sample type has to be specified during the order entry procedure.

The instrument generally works with sample types. Orders have to provide sample information either as a default sample type assignment (specified in the test definition) or with specific sample types.

The Sample Type abbreviation (Lab Type) can be defined in the ‘Configuration/Lab/Sample Type List’ menu.

If Sample Type is enabled in the 'Configuration / Lab / Database Set-up’ menu, this Sample Type abbreviation (Lab Type) is used on the Host Interface.

Note: Ratio sample type (RTO) can also be programmed here for Integra 800 instruments. A code entered here will then override the standard output code RTO with the value give in this table. e.g. "08".  

Note:
If sample-type specific racks are not used, this 'Sample Type abbreviation (Lab Type)' has to be included in the barcode.

 

Barcode set-up

 

Enable barcode reading:

To enable barcode reading, use the ‘Configuration / Lab / Function Enable/Disable’menu.

Supported barcode types:
Code 2 of 5 interleaved
Code 3 of 9 interleaved
Code 2 of 7 (Codabar)
Code 128

The instrument recognizes the barcode type automatically.

 

Configure the racks for barcode reading:

Barcode reading can independently be enabled/disabled for every rack group; configure them in the ‘Configuration / Lab / Rack Configuration / Rack Definition’ menu.

 

 

Sample type reading:

The COBAS Integra 700 offers the possibility of working either with or without sample types
(‘Configuration/Lab/Database Set-up’ menu).
If you selected
- to work with sample type and
- the barcode reader is enabled and
- you don’t use sample type specific racks,
then you must include the sample type in the barcode.

Masking the barcode:

The ‘Configuration/Lab/Sample Coding’ menu defines the set-up of the barcode pattern.

On the Host Interface, sample identification consists of two parts: order number and sample type.
Be aware that they are delivered in two separate fields within the order id. but are combined on the barcode.
If barcode reading is enabled, the order number has to be part of the barcode.

Barcode on sample cups:
Not selected The order entry menu suggests a specific position on a specific rack.
Selected The order entry menu suggests a barcode position.
.
The barcode pattern is coded using:
Order number = O
Sample type = T
Character not used = x
.
Maximum number of characters:
Order number: 15 characters (alphanumeric)
Sample type 3 characters (alphanumeric)
.
Alignment (only applies if the specified pattern is shorter than the barcode used - see example below)
Left = left part of barcode is used
Right = right part of barcode is used
Note:
The barcode must not be shorter
than the pattern you specified.
